dtd约束 （看懂）
	(1)dtd开发步骤
		* 看xml中有多少个元素，有几个创建几个<!ELEMENT>
		* 判断是复杂元素还是简单元素
			- 复杂元素：有子元素的元素
				<!ELEMENT 元素名称 (子元素)>
			- 简单元素：没有子元素
				<!ELEMENT 元素名称 (#PCDATA)>
		* 在xml文件中引入dtd文件
			<!DOCTYPE 根元素名称 SYSTEM "dtd文件路径">
	(2)dtd三种引用方式
		1)外部dtd文件
			<!DOCTYPE 根元素名称 SYSTEM "dtd文件路径">
		2)内部dtd文件
			<!DOCTYPE 根元素名称 [dtd内容]>
		3)引用网上dtd文件
			<!DOCTYPE 根元素名称 PUBLIC "dtd名称" "dtd文档URL">
		
	(3)dtd定义元素
		* 语法<!ELEMENT 元素名 约束>
		* 简单元素：没有子元素
			<!ELEMENT 元素名称 (#PCDATA)>
			- (#PCDATA) 约束内容为字符串类型
			- EMPTY 约束内容为空
			- ANY 任意内容
		* 复杂元素：有子元素的元素
			** <!ELEMENT 元素名 (子元素名)>
				- 子元素只能出现一次
			** 表示子元素出现次数
				- +：1次或多次
				- *：0次或多次
				- ？：0次或1次
			** 子元素用逗号隔开，表示子元素出现顺序(不能改变)
			** 子元素用|隔开，表示元素只能出现其中一个
	
	(4)dtd定义属性
		* 语法 <!ATTLIST 元素名称
					属性名 属性类型 属性约束
				>
	    * 属性类型
			- CDATA 字符串
			- 枚举类型：(aa|bb|cc)枚举列表中选一个
			- ID：值只能是字母或下划线开头
		* 属性约束
			- #REQUIRED：属性必须存在
			- #IMPLIED：属性可有可无
			- #FIXED "":属性值必须用这个指定的固定值
			- 直接值：如果没写属性，默认该元素有这个直接值属性
					  如果写了属性，用设置的值
	(5)dtd定义实体
		* 语法 <!ENTITY 实体名称 "实体内容">
		* 使用方法 &实体名称;
		* 实体应定义在内部dtd中，外部dtd有的浏览器不兼容
